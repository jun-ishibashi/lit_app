## 全輸送方式（航空・海上・鉄道・陸送・複合）向けの拡張

**TRANSPORT_MODES_AND_QUOTE_REQUIREMENTS.md** に、全輸送方式における見積もり・比較に必要な情報をまとめています。

- **共通**：出発地・到着地、重量・容積・寸法・個数、インコタームズ、出荷希望日・希望納期、特殊貨物の有無
- **方式別**：航空（課金重量・FSC/SSC）、海上（FCL/LCL・THC・BAF 等）、鉄道（容積重量係数）、陸送（FTL/LTL）、複合
- **本アプリの現状**：各項目の対応有無の一覧、料金単位の早見表、インコタームズの整理
- **追加機能の優先度**：金額の意味（ALL IN/内訳）、荷物情報（重量・CBM）、インコタームズ、鉄道・陸送・複合の追加、料金単位の多様化 など

金額の単位・通貨の明示は対応済み。

## 優先して進めるとよい順（提案）

1. **テストの充実** … 検索・日付・create・公開プロフィールまわり。環境によっては Minitest の実行方法の見直しが必要な場合あり。
2. **文言の統一** … 「該当するサービス」など、軽い修正で一貫性が上がる。
3. **404/500 ページ** … 本番公開前にあると安心。
4. **ページネーション** … 検索結果が 100 件を超える運用になってからでよい。
5. **見積もり依頼の拡張** … 返信・ステータス更新はユーザー・業者双方の利便性に直結。

その他はフェーズやリソースに応じて ROADMAP の Phase 1〜3 に沿って検討するとよいです。
