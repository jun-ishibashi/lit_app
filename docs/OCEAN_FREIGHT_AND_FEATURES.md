# 海運・国際輸送を踏まえた追加機能の整理（旧版）

> **※ 全輸送方式（航空・海上・鉄道・陸送・複合）を対象にした最新の整理は [TRANSPORT_MODES_AND_QUOTE_REQUIREMENTS.md](TRANSPORT_MODES_AND_QUOTE_REQUIREMENTS.md) を参照してください。**  
> このファイルは海運に特化した旧版で、上記ドキュメントに統合されています。

ネット調査と業界慣行を踏まえ、海運・国際輸送の見積もり・比較に必要な情報と、本アプリで検討すべき追加機能をまとめました。

---

## 1. 見積もり依頼に必要な情報（業界で一般的な項目）

海上輸送の見積もり取得には、おおむね以下の情報が求められます。

| 項目 | 説明 | 現状 |
|------|------|------|
| 輸送方法 | 船・飛行機など | サービスタイプ（航空/海上等）で対応済み |
| 出発地・到着地 | 港・空港の詳細 | 出発地・到着地マスタで対応済み |
| FCL / LCL | フルコンテナか混載か | **未対応**（海上のみで重要） |
| 荷物の形・サイズ・重さ・個数・容積 | 縦×横×高さ、CBM、重量 | **未対応** |
| 商品の内容・注意点 | 品目、危険物等 | 見積依頼の「メッセージ」で代用可能 |
| インコタームズ | EXW, FOB, CIF, DDP 等 | **未対応** |
| 出荷予定日 | いつ出したいか | 検索の「希望出発日」で一部対応 |
| バンニング担当 | コンテナ詰めは誰がするか | **未対応** |
| B/L 形式 | 船荷証券の種類 | **未対応** |
| 特殊貨物 | 危険品・冷蔵・保険対象等 | **未対応** |

## 2. 海運料金の内訳（比較時に重要な観点）

見積もり比較では「総額」だけでなく、**何が含まれているか**を揃えて比べる必要があります。

- **基本運賃（Base Freight）**  
  貨物の種類・重量・容積・距離などで決まる基本の運賃。
- **割増料金（サーチャージ）**  
  - BAF（燃料費調整）、CAF（通貨調整）、YAS（円高損失補填）  
  - PCS（港湾混雑）、PSS（繁忙期） など  
- **港湾・ターミナル関連**  
  - **THC**（ターミナルハンドリングチャージ）：積み地・揚げ地のターミナル取扱料  
  - **CFS チャージ**：混載（LCL）時のコンテナ詰め・取り出し作業料  
- **その他**  
  書類料、保険、ドレージ（陸送）、デマレージ等  

**本アプリでの検討**  
- 料金は現在「1つの数値（円）」のみ。  
- 「ALL IN 価格か」「内訳（基本運賃・THC・サーチャージ等）を持つか」を検討する余地あり。  
- まずは **「表示金額の単位・通貨」を明示**（日本円/JPY）し、必要に応じて「税込/税別」「内訳の有無」を説明欄や別項目で補足する形が現実的。

---

## 3. FCL と LCL（海上輸送の2形態）

| 区分 | 内容 | 料金の考え方 |
|------|------|--------------|
| **FCL**（フルコンテナ） | 1社でコンテナ1本を専有 | 1コンテナあたりの単価（例：20ft 〇〇円） |
| **LCL**（混載） | 複数社で1コンテナを共有 | 容積または重量あたりの単価（例：〇〇円/CBM） |

- 目安：12～15 CBM 以上なら FCL が有利なことが多い。  
- 本アプリでは「サービスタイプ」で航空/海上を分けているが、**海上について「FCL/LCL」や「コンテナサイズ（20ft/40ft）」を区別する項目は未実装**。

---

## 4. インコタームズ（貿易条件）

取引条件により、運賃・保険・リスク・通関の負担が変わります。

- **EXW**（工場渡し）  
- **FOB**（本船渡し）  
- **CIF**（運賃・保険込み）  
- **DDP**（輸入通関込み・届け先渡し） など  

見積もり依頼時にインコタームズを指定できると、**同じ条件で業者間を比較**しやすくなります。現状は未対応。

## 5. 追加機能の優先度案

以下は「海運・国際輸送」を意識した追加機能の整理です。実装コストと効果を考慮し、優先度を付けてあります。

### 高：金額の単位・通貨の明示 【対応済み】

- 表示・入力が「日本円（JPY）」であることを明示。  
- 一覧の「料金」列を「料金（円）」に、詳細では「（日本円）」を表示。  
- 将来的に多通貨対応する場合は、`currency` カラムや「税込/税別」フラグの追加を検討。

### 高：料金の意味の明示（ALL IN / 内訳）

- サービス登録時に「この料金に含まれるもの」を選択または短文で入力できるようにする。  
  - 例：「基本運賃のみ」「基本運賃+THC」「ALL IN（諸経費込み）」  
- または説明欄で「税別」「通関含む」等を記載する運用で補足。

### 中：海上の輸送形態（FCL / LCL）

- サービスタイプが「海上」のとき、**FCL / LCL** を選択可能にする。  
- FCL の場合は **コンテナサイズ**（20ft / 40ft 等）を選択できると、比較しやすい。

### 中：料金の単位の多様化（海上向け）

- 海上 LCL：**円/CBM** や **円/重量** で登録できるようにする。  
- 海上 FCL：**円/コンテナ** のままでもよいが、「20ft あたり」「40ft あたり」を明示。  
- 現行の「円」1本のサービス料金と併存させる場合は、  
  - `price_type`（例：total / per_cbm / per_container）  
  - `price_unit`（CBM, kg, 20ft, 40ft）  
  などの項目追加を検討。

### 中：インコタームズ

- サービスまたは見積依頼時に **インコタームズ**（EXW, FOB, CIF, DDP 等）を選択可能にする。  
- マスタは ActiveHash または DB で保持。  
- 検索・一覧で「同じインコタームズで絞る」があると比較しやすい。

### 中：荷物情報（容積・重量・個数）

- 見積もり依頼時に **CBM（容積）・重量・個数** を入力できるようにする。  
- 業者側が「〇 CBM 以下ならこの単価」などと登録している場合、検索・マッチングに利用できる。

### 低：出荷予定日・希望納期

- 検索で「希望出発日」「希望到着日」は一部対応済み。  
- 「出荷予定日」を依頼時に必須にしたり、業者側の「対応可能な出荷時期」を登録する機能は将来検討。

### 低：サーチャージ・内訳の構造化

- 基本運賃・BAF・THC・CFS 等を**項目別に登録**する仕様は、業界では有用だが実装負荷は大。  
- まずは「ALL IN かどうか」「説明欄で内訳を記載」で対応し、必要に応じて構造化を検討。

### 低：特殊貨物・B/L 形式・バンニング

- 危険品・冷蔵・保険、B/L 形式、バンニング担当は、  
  まずは見積依頼の「メッセージ」やサービス「説明」で対応し、  
  需要に応じて選択肢やマスタを追加。

## 6. まとめ

| 観点 | 推奨アクション |
|------|----------------|
| **金額の単位** | 日本円（JPY）であることを画面で明示（表ヘッダ「料金（円）」・詳細で「日本円」表記）→ **対応済み** |
| **料金の意味** | 「税込/税別」「ALL IN か」を説明または選択で補足できるようにする |
| **海運の比較** | FCL/LCL・コンテナサイズ・インコタームズを段階的に追加 |
| **見積もり精度** | 依頼時の CBM・重量・個数があると、業者側の単価と組み合わせやすい |
| **内訳** | 初期は説明欄で対応し、必要に応じて項目別（基本運賃・THC 等）を検討 |
