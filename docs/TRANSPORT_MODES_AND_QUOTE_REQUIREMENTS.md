# 全輸送方式における見積もり・比較に必要な情報

海運に限らず、**航空・海上・鉄道・陸送・複合**など、輸送方法全てを対象に、業界慣行とネット調査に基づき「見積もりに必要な情報」と「料金の考え方」をまとめました。本アプリで検討すべき追加機能の優先度案も記載しています。

---

## 1. 共通で必要な情報（全輸送方式）

見積もり依頼時に、どの輸送方式でも共通して求められる情報です。

| 項目 | 説明 | 本アプリの現状 |
|------|------|----------------|
| **輸送方法** | 航空 / 海上(FCL/LCL) / 鉄道 / 陸送 / 複合 | サービスタイプ（航空・船舶LCL・船舶FCL）で対応済み。鉄道・陸送・複合は未対応 |
| **出発地・到着地** | 港・空港・駅・住所など、できるだけ具体まで | 出発地・到着地マスタで対応済み |
| **荷物の重量** | 実重量（kg） | **未対応**（依頼時に未入力） |
| **荷物の容積・寸法** | 縦×横×高さ（cm）、CBM（m³） | **未対応** |
| **個数・梱包形態** | 箱数、パレット、裸梱包など | **未対応** |
| **品目・内容** | 商品説明、HSコード（任意） | 見積依頼の「メッセージ」で代用可能 |
| **出荷希望日・希望納期** | いつ出したいか・いつまでに届けたいか | 検索の希望出発日・希望到着日で一部対応 |
| **インコタームズ** | EXW, FOB, CIF, DAP, DDP 等 | **未対応** |
| **ドア〜ドアか港/空港引取か** | 最終配送範囲 | 作業範囲（ServiceScope）で一部表現可能 |
| **特殊貨物** | 危険品、冷蔵、高価品（保険）の有無 | **未対応**（メッセージで補足） |

## 2. 輸送方式別の料金の決まり方と必要情報

### 2.1 航空輸送（Air Freight）

**課金重量（チャージャブルウェイト）**

- **実重量**：実際の重さ（kg）
- **容積重量**：縦×横×高さ（cm）÷ **6,000** で計算
- **課金対象**：実重量と容積重量の**いずれか大きい方**

見積もりに必要な情報の例：

- 実重量（kg）、寸法（L×W×H cm）
- 出発・到着空港（または都市）
- 貨物の種類（一般 / 危険物等）
- 納期（緊急か通常か）
- 経由地の指定の有無

**料金内訳の例**

- 基本運賃（重量×レート）
- 燃油サーチャージ（FSC）
- セキュリティチャージ（SSC）
- ターミナルハンドリング、書類料、現地諸掛かり

比較のポイント：総額だけでなく「何が含まれているか」を揃えて比較する。

---

### 2.2 海上輸送（Ocean Freight）

**FCL（フルコンテナ）**

- 料金は**1コンテナあたり**（20ft / 40ft 等で単価が異なる）
- 見積に必要な情報：コンテナサイズ、出発港・揚げ港、出荷希望日、インコタームズ、ドア配送かCY/CFS引取か

**LCL（混載）**

- 料金は**容積（CBM）または重量**のいずれか大きい方で計算することが多い（業者により係数は異なる）
- 見積に必要な情報：CBM（または寸法＋重量）、出発港・揚げ港、品目、インコタームズ

**海上の料金内訳の例**

- 基本運賃（Ocean Freight）
- BAF（燃料費調整）、CAF（通貨調整）、YAS（円高損失補填）等のサーチャージ
- THC（ターミナルハンドリングチャージ）：積み地・揚げ地
- CFSチャージ（LCL時のコンテナ詰め・取り出し）
- 書類料、ドレージ（陸送）、デマレージ等

**海上特有の検討項目**

- バンニング（コンテナ詰め）の担当
- B/L（船荷証券）の形式（Sea Waybill 等）

---

### 2.3 鉄道輸送（Rail Freight）

例：中欧班列（中国〜欧州）、ユーラシア陸路など。

**課金の考え方**

- 実重量と**容積重量**のいずれか大きい方で計算する場合が多い
- 鉄道の容積重量係数は業者・ルートにより異なる（例：DHL では 縦×横×高さ cm ÷ **3,000** で容積重量 kg を算出する方式など）
- コンテナ単位で見積もる場合は、20ft/40ft 等のサイズと本数が前提

**見積に必要な情報の例**

- 重量（kg）、容積（CBM）または寸法
- 出発地・目的地（駅または都市）
- コンテナサイズ（FCL の場合）、または LCL の CBM
- 品目、特殊扱いの有無

**料金・納期の目安（参考）**

- 中国〜欧州：15〜20日程度、コンテナ単価で変動
- ルート・時期により価格変動が大きいため、見積もり取得が必須

---

### 2.4 陸上輸送（Road / Truck）

国際的な陸送（国境をまたぐトラック輸送）や、港・空港からの内陸配送（ドレージ）が該当。

**料金の決まり方**

- 距離、重量・容積、荷物の種類、ピックアップ/デリバリーの範囲（ドア〜ドアかターミナルのみか）
- FTL（Full Truck Load）/ LTL（Less Than Truck Load）の区別がある場合あり

**見積に必要な情報の例**

- ピックアップ先・配送先の住所（郵便番号・国・都市）
- 重量・容積・パレット数
- 積み下ろしの条件、荷役の要否

---

### 2.5 複合輸送（Multimodal）

海＋鉄道、航空＋陸送など、複数の輸送手段を組み合わせた一貫輸送。

**見積に必要な情報**

- 上記の「共通項目」に加え、**最終的な受け渡し条件**（ドア〜ドアか、港/空港/駅渡しか）を明確にする
- インコタームズ（DAP, DDP 等）が重要
- 区間ごとの責任範囲と、総額に何が含まれるか（通関・保険・内陸運賃の有無）を確認

---

### 2.6 国際小包・宅配（参考）

航空便・船便・SAL など、小口の国際郵便や国際宅配。

**見積に必要な情報**

- 送り先国・地域（地帯で料金が異なる）
- 重量（例：最大 30kg まで等、国により制限あり）
- サイズ（最長辺・長さ+周囲の合計など、基準が国ごとにある）
- 配送方法（航空 / 船便 / SAL 等）

本アプリの対象が「BtoB の貨物・フォワーダー見積もり」であれば、小包は対象外でよいが、参考として記載。

## 3. 輸送方式別「課金重量・料金単位」早見表

| 輸送方式 | 課金の考え方 | 料金単位の例 | 備考 |
|----------|--------------|--------------|------|
| **航空** | 実重量 vs 容積重量（÷6,000）の大きい方 | 円/kg または 総額 | チャージャブルウェイト |
| **海上 FCL** | コンテナ1本あたり | 円/20ft, 円/40ft | サイズ別単価 |
| **海上 LCL** | CBM または重量の大きい方（係数は業者による） | 円/CBM または 円/kg | 最低料金ありのことが多い |
| **鉄道** | 実重量 vs 容積重量（係数は業者による。例 ÷3,000） | 円/kg または コンテナ単価 | ルート・時期で変動大 |
| **陸送** | 距離・重量・容積・車両占有 | 円/便、円/km 等 | FTL/LTL で異なる |
| **複合** | 区間ごとに上記を組み合わせ | 総額（ALL IN 表示が多い） | インコタームズで範囲を明確に |

## 4. インコタームズ（貿易条件）の整理

見積もり・比較では「誰がどこまで責任と費用を負うか」を揃えるため、**インコタームズ**の指定が重要です。

| 条件 | 主な意味 | 主な用途 |
|------|----------|----------|
| **EXW** | 工場渡し（買主がすべて手配） | 輸出者負担を最小にしたい場合 |
| **FCA** | 指定地で運送人に渡し | 複合輸送でよく使う |
| **FOB** | 本船積込渡し | 海上のみ、買主が運賃・保険を手配 |
| **CFR/CIF** | 運賃込み / 運賃・保険込みで輸入港まで | 海上、港までの費用を売主負担に |
| **CPT/CIP** | 運賃込み / 運賃・保険込み（複合輸送向け） | 航空・複合で使う |
| **DAP** | 指定地持込渡し（輸入通関・関税は買主） | 全モード対応 |
| **DDP** | 指定地まで関税込で持込渡し | 買主負担を最小にしたい場合 |

見積依頼時に「FOB」「DDP」などを選べると、**同じ条件で業者間を比較**しやすくなります。

## 5. 本アプリで検討すべき追加機能（優先度付き）

以下は、**全輸送方式**を踏まえた追加機能の整理です。

### 高：金額の単位・通貨の明示 【対応済み】

- 表示を「料金（円）」、詳細で「日本円（JPY）」と明示済み。
- 将来的に多通貨対応する場合は `currency` カラムや税込/税別フラグの追加を検討。

### 高：料金の意味の明示（ALL IN / 内訳）【対応済み】

- サービス登録・編集フォームに「料金に含まれるもの」欄（`price_includes`）を追加。短文で入力可能（例：ALL IN、税別、基本運賃+THC）。
- サービス詳細画面で表示。

### 高：荷物情報（重量・容積・寸法・個数）【対応済み】

- 見積もり依頼フォームに **重量（kg）・容積（CBM）・個数** を追加。依頼詳細で表示。

### 中：インコタームズ【対応済み】

- **Incoterm**（ActiveHash）を追加。サービス登録・見積依頼でインコタームズを選択可能。検索で「インコタームズ」絞り込みに対応。

### 中：輸送方式の拡張（鉄道・陸送・複合）【対応済み】

- **ServiceType** に 鉄道・陸送・複合 を追加。サービスタイプの選択肢と検索で利用可能。

### 中：料金単位の多様化【対応済み】

- サービスに **price_type**（total / per_kg / per_cbm / per_container）と **price_unit**（20ft, 40ft）を追加。一覧・詳細で「30,000円」「500円/kg」「25,000円/20ft」のように表示。

### 中：海上の補足項目（FCL/LCL は既存サービスタイプで対応済み）【対応済み】

- **ContainerSize**（ActiveHash: 20ft, 40ft）を追加。サービス登録時にコンテナサイズを選択可能。詳細・検索で表示・絞り込み対応。

### 低：出荷希望日・希望納期の強化

- 検索の希望出発日・希望到着日は一部対応済み。
- 依頼時に「出荷希望日」「希望納期」を必須または推奨項目にし、業者側の「対応可能な出荷時期」を登録できるようにするのは将来検討。

### 低：サーチャージ・内訳の構造化

- 基本運賃・FSC/BAF・THC・書類料等を**項目別に登録**する仕様は、業界では有用だが実装負荷は大。
- まずは「ALL IN かどうか」「説明欄で内訳を記載」で対応し、必要に応じて構造化を検討。

### 低：特殊貨物・書類

- 危険品、冷蔵、高価品（保険）は、まずは見積依頼の「メッセージ」やサービス「説明」で対応。
- B/L 形式、バンニング担当、MSDS 等の要否は、需要に応じて選択肢やマスタを追加。

- [ ] 輸送方法（航空 / 海上FCL / 海上LCL / 鉄道 / 陸送 / 複合のいずれか）
- [ ] 出発地・到着地（港・空港・駅・住所など、できるだけ具体）
- [ ] 荷物の重量（kg）
- [ ] 荷物の寸法（縦×横×高さ cm）または容積（CBM）
- [ ] 個数・梱包形態
- [ ] 品目・内容（危険物の有無）
- [ ] 出荷希望日・希望納期
- [ ] インコタームズ（希望する取引条件）
- [ ] ドア〜ドア希望か、港/空港/駅引取か
- [ ] 予算・スピードの優先度（コメントで伝えるとよい）

## 7. まとめ

| 観点 | 推奨アクション |
|------|----------------|
| **共通** | 出発地・到着地は対応済み。**重量・容積・寸法・個数**を依頼時に取れるようにすると、全方式で見積精度が向上する。 |
| **金額** | 単位・通貨の明示は対応済み。**料金の意味**（ALL IN / 内訳、税込/税別）を説明または選択で補足できるようにする。 |
| **方式別** | 航空は課金重量、海上は FCL/LCL・コンテナサイズ、鉄道は容積重量係数やコンテナ単位を考慮。必要に応じて**鉄道・陸送・複合**をサービスタイプに追加。 |
| **比較** | **インコタームズ**と**荷物情報**を揃えると、業者間の比較がしやすい。 |
| **内訳** | 初期は説明欄で対応し、必要に応じて項目別（基本運賃・サーチャージ等）の構造化を検討。 |

上記を `REMAINING_ITEMS.md` や `ROADMAP` と連携させ、スプリント単位で取り込むと整理しやすいです。
