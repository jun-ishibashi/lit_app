# CargoLink — マネタイズの可能性

海外輸出の輸送比較プラットフォームとして、**マネタイズできる可能性はある**。ただし「比較だけ」の段階では限定的で、**依頼側と業者をつなぐ機能（問い合わせ・見積・成約の可視化）** ができるほど、収益の選択肢が増える。

---

## 結論


| 観点           | 内容                                                           |
| ------------ | ------------------------------------------------------------ |
| **可能性**      | あり。BtoB の「比較・マッチング」系サービスとして、業者課金・紹介報酬・サブスクなどが現実的な候補。         |
| **前提**       | 依頼側の利用数（検索・比較）と業者の掲載数が一定以上あること。まずは「使われるサービス」にすることが優先。        |
| **向いている収益源** | 業者（運送業者）からの課金。依頼側は無料のまま、業者に「掲載料・リード課金・プレミアム表示」などを求める形が取りやすい。 |


---

## 想定できる収益モデル

### 1. 業者向け課金（掲載・リード・プレミアム）


| モデル         | 内容                                    | 実現しやすさ | 備考                                                      |
| ----------- | ------------------------------------- | ------ | ------------------------------------------------------- |
| **掲載料・月額**  | サービスを掲載する業者に月額または年額を課金。               | ◎      | 掲載業者数が増え、依頼側の検索が増えるほど説明しやすい。無料プラン（掲載本数制限）＋有料プランで段階設定も可。 |
| **リード課金**   | 「相談する」「見積依頼」などで業者にリード（問い合わせ）が届くたびに課金。 | ◎      | 相談・問い合わせ機能の実装が前提。成約に直結しやすいため業者が負担を理解しやすい。               |
| **プレミアム表示** | 検索結果の上位表示・バッジ・強調表示を有料オプションに。          | ○      | 検索利用が増えてから意味が出る。広告モデルに近い。                               |


**ポイント**: 依頼側（ユーザー）は**無料**にしておくと、利用が広がりやすく、その結果として業者側の課金を正当化しやすい。

---

### 2. 成約・取引に紐づく報酬


| モデル          | 内容                                          | 実現しやすさ | 備考                                              |
| ------------ | ------------------------------------------- | ------ | ----------------------------------------------- |
| **成功報酬・紹介料** | プラットフォーム経由で成約した場合、運賃の数％や一定額を業者または依頼側から受け取る。 | △      | 成約を「CargoLink 経由」と証明する仕組み（申込フロー・契約連携）が必要。業界の慣行次第。 |
| **決済手数料**    | 将来的に代金決済を CargoLink 上で行う場合、手数料を取る。            | △      | 実装・法規制・与信のハードルが高く、中期以降の検討向き。                    |


**ポイント**: 現状は「検索・比較」のみなので、まずは**問い合わせ・見積依頼**までつなぎ、その先の成約をどう計測するかを決める必要がある。

---

### 3. 依頼側（ユーザー）向け課金


| モデル         | 内容                             | 実現しやすさ | 備考                                       |
| ----------- | ------------------------------ | ------ | ---------------------------------------- |
| **プレミアム会員** | 詳細検索・保存・複数見積一括依頼・優先サポートなどを有料に。 | △      | 無料でも価値が分かる状態になってから。差別化する機能の設計が要。         |
| **依頼側の手数料** | 見積依頼1件あたりの手数料など。               | △      | 業界によっては依頼側が手数料に敏感なため、業者課金の方が導入しやすい場合が多い。 |


**ポイント**: まずは**依頼側は無料**で集客し、**業者課金**で収益化する方が現実的。

---

### 4. その他


| モデル          | 内容                             | 備考            |
| ------------ | ------------------------------ | ------------- |
| **広告**       | 関連サービス（倉庫・通関・保険など）のバナー・リスティング。 | トラフィックが増えてから。 |
| **データ・レポート** | ルート別の相場・トレンドを業界向けに有料提供。        | データが溜まってから。   |

## マネタイズが現実的になる条件

1. **依頼側の利用が一定以上ある**
  検索・比較が「当たり前に使われる」状態。業者にとって「ここに載せると問い合わせが来る」と感じられること。
2. **業者が「掲載する意味」を感じている**
  問い合わせやリードが届く設計（相談する・見積依頼）が動いていること。掲載料・リード課金はここから説明しやすい。
3. **信頼と実績**
  業者審査・評価・実績の見える化があると、依頼側も業者も課金に納得しやすい。
4. **競合・ポジション**
  既存の貨物比較・フォワーダー紹介サービスとの差別化（UX・料金の見え方・業者数・対象ルートなど）がはっきりしていると、有料化を打ち出しやすい。

---

## ロードマップとの関係

- **Phase 1 まで**: 基盤と信頼性の整備。マネタイズより「使われるサービス」にすることが中心。
- **Phase 2**: 相談・問い合わせの実装で**リードが発生**。ここから「業者向けリード課金」「掲載料」の検討が現実的になる。
- **Phase 3**: 成約・評価・履歴が貯まれば、**成功報酬・プレミアム表示・データ販売**なども選択肢に入る。

---

## まとめ

- **マネタイズの可能性はある**。特に**業者向けの掲載料・リード課金・プレミアム表示**が現実的な候補。
- 依頼側は当面無料にし、**「相談する」などで依頼側と業者をつなぐ機能**を実装することが、収益化の第一歩になる。
- まずは Phase 2 の「相談・問い合わせ」と利用数の確保を進め、その上で業者向けの有料プラン・リード課金の具体案を詰めるとよい。
