# CargoLink — 目的・ターゲット・指標（公式定義）

このドキュメントは、CargoLink の**公式なポジションと成功の測り方**を一括で定義したものです。README・ROADMAP・画面コピー・機能優先度は、ここに合わせて揃えます。

---

## 1. ミッション・背景

### 1.1 ミッション（一文）

**「物流・海外貿易に詳しくない人でも、損せず正しく判断できるようにする。」**

- 航空・海運・鉄道など、**どの手段を選ぶべきか**を適切に比較できるようにする。
- **業者の選定**と**輸送手段の選定**の両方を、専門家でなくても適切に行えるようにする。

### 1.2 背景・課題

- 中小企業・個人事業主には、物流に詳しい人がいないことが多い。
- 「とにかく見積もりを依頼するしかない」状態になりがちで、**高く見積もられたり**、**手段や業者を誤って選んでしまう**リスクがある。
- CargoLink は、**各手段を適切に比較できる**ことで、詳しくない人でも業者選定・手段選定を正しく行えるようにする。

### 1.3 ミッションから導く方針

| 観点 | 方針 |
|------|------|
| **ターゲット** | 物流・海外貿易に詳しくない人（中小企業・個人事業主・初めて輸出する法人など）を主な利用者とする。 |
| **提供価値** | 航空・海運・鉄道などを「同じ条件で並べて比較できる」ことを前面に出す。手段の違い（早い／安い／量に合う）がわかるようにする。 |
| **判断の支援** | 見積もり依頼の前に「どれを選ぶか」を判断できる情報（料金・リードタイム・手段の特徴）を提供する。 |

## 2. 採用したターゲット・範囲（最初に狙うスライス）

体験と認知をまず作る範囲を以下に固定する。広げるのは、このスライスで「比較が効いている」と実感できるようになってからとする。

| 軸 | 内容 | 備考 |
|----|------|------|
| **方向** | **日本→海外の輸出**を主軸 | 輸入・第三国間は検索・データ上は対応するが、トップや説明では「輸出で送りたい人向け」を前面に出す。 |
| **輸送手段** | **航空・海上**を前面に | 鉄道・陸送・複合は検索では選べるが、訴求コピー・選び方ガイドは「航空 vs 海上」を中心にする。 |
| **地域** | **日本↔アジア・北米・欧州**など主要ルート | 出発地・到着地は全マスタを表示するが、事例・説明・SEO では東南アジア・アメリカ・欧州などよく使うルートを優先して見せる。 |
| **依頼側** | **中小企業・個人事業主・初めて輸出する法人** | 個人の輸出・小口発送も対象に含めるが、文言では「物流に詳しくない事業者」を主に想定する。 |
| **掲載者** | **フォワーダー（国際貨物取次業者）**を想定 | キャリア直結や専門業者の掲載は運用で判断。業者向けには「物流に不慣れな層にリーチできる」「条件が合う依頼が来る」をメリットとして打ち出す。 |

**運用**: トップ・README・業者向けページのコピーは上記に合わせる。機能は鉄道・陸送・複合や全ルートを残したまま、「まずは航空・海上で輸出を比較」というメッセージに揃える。

## 3. 成功指標と集計方法

### 3.1 採用している指標

| 指標 | 内容 | 用途 |
|------|------|------|
| **検索で 2 件以上を比較した割合** | 検索実行のうち、結果が 2 件以上表示された検索の割合。 | 「複数案を見て選んでいる」行動の代理指標。 |
| **見積もり依頼を 2 社以上にした割合** | 1 件以上依頼したユーザーのうち、異なる業者に 2 社以上依頼したユーザーの割合。 | 「相見積もりを取れている」＝損しにくい判断に近づいているか。 |
| **見積もり依頼数（月・累計）** | 月あたり・累計の見積もり依頼件数。 | 依頼側の利用度と、業者にとってのリード数。 |
| **掲載業者数・掲載サービス数** | 1 件以上サービスを登録している業者数、サービス数。 | 比較の土台が十分かどうか。 |
| **検索実行数（月）** | 月あたりの検索実行回数。 | 認知・流入と「比較しようとしている」行動量。 |

**将来追加したい指標**: 成約数（CargoLink 経由と分かる場合）、成約後・見積依頼後の満足度。

### 3.2 集計方法（データソース・算出）

| 指標 | データソース | 集計方法 |
|------|--------------|----------|
| 検索で 2 件以上比較した割合 | 検索ログ（要追加） | 期間内の検索のうち、結果件数が 2 件以上だった検索の割合。SearchLog で結果件数・日時を保存して集計。 |
| 見積もり依頼を 2 社以上にした割合 | 既存 DB（`quote_requests`） | 期間内に 1 件以上依頼したユーザーのうち、異なる `provider_id` に 2 社以上依頼したユーザーの割合。既存データのみで算出可能。 |
| 見積もり依頼数（月・累計） | 既存 DB | `QuoteRequest.where(created_at: 期間).count`。 |
| 掲載業者数 | 既存 DB | `Provider.joins(:services).distinct.count`（1 件以上サービスを持つ業者）。 |
| 掲載サービス数 | 既存 DB | `Service.count`。 |
| 検索実行数（月） | 検索ログ（要追加） | 期間内の SearchLog 件数。未導入時はアクセスログで `/services/search` の GET を数える方法も可。 |

**検索ログの最小設計**

- テーブル: `search_logs`。カラム例: `id`, `results_count`（integer）, `created_at`。
- 記録: `ServicesController#search` 実行後、`SearchLog.create!(results_count: @services.size)` で 1 行保存。
- 必要に応じて `user_id` や `session_id` を追加可能。

**進め方**: まず見積もり依頼数・掲載業者数・掲載サービス数・2 社以上依頼割合を既存 DB で週次・月次把握。SearchLog 導入後に検索実行数・2 件以上比較割合を追加。月次サマリを管理画面やスプレッドシートで共有する。

## 付録 A: ミッションの評価・チャンス・注意点

**良い点**: 課題（詳しくない→見積もり一本→高くつく・選び損ない）が明確で、解決策（比較できるようにする）と一対一で結びついている。ターゲットが「詳しくない人」に限定されているため、機能・文言を専門家向けに振り切らなくてよい。手段選定と業者選定の両方を扱うポジションが伝わりやすい。

**注意点**: 「損しない」は「いちばん安い」だけでなく、納期・信頼性・通関などを踏まえた「自分の状況に合った選択」と定義しておくとよい。依頼側への課金はミッションと説明が要るため、業者課金（掲載料・リード課金）の方が整合しやすい。

**チャンス**: 「海外貿易をやりたいが物流は素人」な中小・個人向けに、複数手段・複数業者を同じ土俵で比較できるサービスは日本ではまだ少ない。輸出挑戦・小口輸出の増加で比較ニーズはある。差別化を「比較のわかりやすさ」と「手段選定の支援」に絞り、信頼と掲載数をつくっていくことが重要。業者には「質の高い問い合わせが届く」「初めての輸出者にリーチできる」をメリットとして打ち出す。

## 付録 B: 検討の経緯（参考）

本ドキュメントの定義を決めるにあたり、以下の「ぶれ」を整理した。

- **範囲**: 文言は「航空・海上」中心だったが、実装では鉄道・陸送・複合も存在 → 訴求を「航空・海上を前面に」とし、検索では全モードを残す形で統一。
- **ターゲット**: 「生産者・中小企業」と「個人・多品目」が混在 → 「物流に詳しくない人（中小企業・個人事業主・初めて輸出する法人）」を主とし、個人も含める形で統一。
- **方向**: 「海外輸出」のみ言及されていたが、出発地・到着地は汎用 → 「日本→海外の輸出を主軸」とし、輸入・第三国間は検索では対応するが訴求では前面に出さない形で統一。
- **掲載者**: 「運送業者」のみでフォワーダー／キャリアの区別が未整理 → 「フォワーダーを想定」とし、業者向けメリットを明文化。

複数のポジション案（輸出特化・航空海上メイン／全モード・幅広い依頼側／生産者・一次産業寄り）を比較したうえで、現在の「最初に狙うスライス」（§2）を採用した。
